## 一、系统概述

本系统是一个基于 **Qt6 + C++ / OpenCV / YOLOv8 / ONNX Runtime** 开发的桌面应用程序，包含以下四大核心功能模块：

| 功能模块 | 说明 |
|---------|------|
| **图像标注** | 对图片进行矩形框标注，支持选中/移动/修改类别/删除，保存YOLO格式标签 |
| **YOLOv8训练** | 调用Python ultralytics库训练目标检测模型，实时显示训练进度 |
| **模型量化** | 将ONNX模型进行INT8/UINT8动态量化压缩，减小模型体积 |
| **推理分析** | Python(onnxruntime)或C++(OpenCV DNN)加载量化模型进行表情分类推理 |

## 二、环境配置（首次使用必须完成！）

### 2.1 Python环境

本系统的训练、量化、Python推理功能依赖Python环境和以下库：

```bash
# 如果使用conda（推荐）
conda create -n zyx python=3.9
conda activate zyx

# 安装必要依赖
pip install ultralytics onnxruntime opencv-python

# 如果需要模型转换功能（可选）
pip install tensorflow tf2onnx onnx
```

**⚠️ 重要提示**：
- 系统启动时会自动检测Python环境，在右上角显示检测结果
- 如果显示 **"Python: ❌ 未找到ultralytics环境"**，请先安装依赖再使用训练/量化功能
- 安装完成后，重启本程序即可自动重新检测

### 2.2 Qt开发环境

- Qt 6.x (macOS/Windows/Linux)
- Qt模块依赖：Core、Gui、Widgets、Sql（SQLite）
- OpenCV 4.x（含dnn模块）

### 2.3 数据集准备

训练前需要准备好数据集，数据集目录结构应包含：
- 图片文件（jpg/png/bmp等格式）
- 对应的YOLO格式标签文件（同名.txt文件）

标签文件格式（YOLO格式，每行一个标注框）：
```
类别ID 中心X 中心Y 宽度 高度
```
其中坐标值为归一化的（0~1之间），例如：
```
0 0.5 0.5 0.3 0.4
```

## 三、功能操作指南

### 3.1 图像标注

#### 3.1.1 加载图片

1. 点击 **"打开图片文件夹"** 按钮（或菜单：文件→打开图片文件夹，快捷键 Ctrl+O）
2. 选择包含图片的文件夹
3. 图片列表会显示在右侧面板中，双击可切换图片

#### 3.1.2 标注操作

| 操作 | 方法 |
|------|------|
| **画标注框** | 左键拖拽绘制矩形框 |
| **选中标注框** | 点击已有标注框（白色高亮表示选中） |
| **移动标注框** | 选中框后拖拽移动位置 |
| **修改框类别** | 选中框 → 在下拉框选择新类别 → 点击"修改框类别"按钮 |
| **删除标注框** | 选中框 → 点击"删除选中框"按钮 / 按 Delete键 / 右键菜单 |
| **视图缩放** | Ctrl+滚轮 或 点击放大/缩小按钮 |
| **视图平移** | Ctrl+左键拖拽 或 中键拖拽 |
| **旋转图片** | 点击↺左转/右转↻按钮 |

#### 3.1.3 类别设置

在标签类别输入框中设置类别名称，格式为：
- `0:angry,1:disgust,2:fear,4:happy,5:sad,6:surprise,7:neutral`（编号:名称格式）
- 或 `angry,disgust,fear,happy,sad,surprise,neutral`（仅名称格式）

#### 3.1.4 保存标注

标注会自动保存，格式为YOLO归一化坐标。保存位置：
- 文件保存：`图片同名.txt`，位于图片文件夹中
- 数据库保存：SQLite数据库 `annotation_data.db`

### 3.2 YOLOv8训练

#### 3.2.1 训练步骤

1. **设置数据集路径**：在"训练配置"区域输入或浏览选择数据集目录
2. **设置训练参数**：
   - Epochs：训练轮数（默认50，可调1~1000）
   - Batch Size：批次大小（默认16，根据GPU内存调整）
3. **设置类别名称**：在标签类别输入框中填写训练类别
4. **点击"开始YOLOv8训练"按钮**

#### 3.2.2 训练过程

- 训练开始后，状态标签显示 "⏳ 训练中..."
- 进度条实时更新当前epoch进度
- 底部日志区实时显示训练输出（深色背景，易于阅读）
- **❌ 如果训练失败**：日志区会红色高亮显示错误原因，弹窗提示具体错误

#### 3.2.3 训练结果

训练成功后：
- 模型权重保存在 `数据集目录/yolo_dataset/runs/detect/train/weights/best.pt`
- 日志区绿色显示成功信息和模型路径
- 训练记录自动保存到数据库（可通过"训练记录"按钮查看）

#### 3.2.4 常见训练错误

| 错误现象 | 原因 | 解决方法 |
|---------|------|---------|
| "ModuleNotFoundError: No module named 'ultralytics'" | Python环境缺少ultralytics | `pip install ultralytics` |
| 训练立即结束，弹窗显示失败 | 数据集路径错误或无图片 | 检查数据集目录是否包含图片 |
| 训练中断退出码非0 | CUDA错误或其他异常 | 查看日志区的stderr信息 |

### 3.3 模型量化

#### 3.3.1 量化步骤

1. **选择ONNX模型**：点击"浏览"按钮选择训练后导出的ONNX模型
2. **选择量化类型**：QInt8（有符号整型）或 QUInt8（无符号整型）
3. **点击"开始量化"按钮**

#### 3.3.2 量化说明

- 量化过程使用ONNX Runtime的动态量化功能
- 量化后模型大小通常减少到原来的25%~50%
- 量化完成后，推理模型路径会自动更新为量化后的模型
- 输出文件命名：原文件名_quantized.onnx

#### 3.3.3 获取ONNX模型的方法

1. **从YOLOv8训练结果导出**：训练完成后，使用ultralytics导出
   ```python
   from ultralytics import YOLO
   model = YOLO("best.pt")
   model.export(format="onnx")
   ```
2. **从Keras HDF5模型转换**：使用本程序的convert_onnx.py脚本
   ```bash
   python convert_onnx.py --input fer2013_mini_XCEPTION.102-0.66.hdf5 --output emotion.onnx
   ```

### 3.4 推理分析

#### 3.4.1 推理步骤

1. **加载图片**：先打开包含待检测图片的文件夹
2. **选择推理模型**：点击"浏览"选择量化后的ONNX模型（.onnx文件）
3. **选择推理引擎**：
   - **Python (onnxruntime)**：需要Python环境和onnxruntime库
   - **C++ OpenCV DNN**：无需Python，直接在C++端使用OpenCV加载模型推理
4. **点击"执行推理"按钮**

#### 3.4.2 推理结果

- 推理结果显示在"结果"标签中
- 成功：绿色背景显示分类结果和置信度
- 失败：红色背景显示错误信息
- 所有推理记录自动保存到数据库

#### 3.4.3 两种推理引擎对比

| 特性 | Python推理 | C++推理 |
|------|-----------|---------|
| 依赖 | Python + onnxruntime | 仅OpenCV（已内置） |
| 速度 | 较慢（进程启动开销） | 较快（本地直接推理） |
| 模型兼容性 | 更广泛 | 部分ONNX算子可能不支持 |
| 推荐场景 | 模型测试 | 生产部署 |

## 六、数据库说明

系统使用SQLite数据库（`annotation_data.db`）保存三类数据：

| 数据表 | 内容 |
|--------|------|
| training_records | 训练参数、模型路径、训练时间 |

## 七、常见问题 FAQ

### Q1: 启动后显示 "Python: ❌ 未找到ultralytics环境"

**解决方法**：
1. 确认已安装conda或Python3
2. 创建/激活虚拟环境：`conda activate zyx`
3. 安装依赖：`pip install ultralytics onnxruntime opencv-python`
4. 重启本程序

### Q2: 点击训练按钮后立即弹窗显示失败

**原因**：Python环境中缺少ultralytics模块，或数据集路径无效。
**解决**：先看日志区的红色错误信息，按提示安装依赖或修正路径。

### Q3: 训练完成但找不到模型文件

模型保存在 `数据集目录/yolo_dataset/runs/detect/train/weights/` 下：
- `best.pt` — 最佳模型（mAP最高的epoch）
- `last.pt` — 最后一个epoch的模型

### Q4: 量化失败

**原因**：缺少onnxruntime库，或ONNX模型文件损坏。
**解决**：`pip install onnxruntime`，并确保输入的ONNX文件是有效的。

### Q5: C++推理显示异常

**原因**：OpenCV DNN模块不支持该ONNX模型的某些算子。
**解决**：切换到Python推理模式（选择"Python (onnxruntime)"）。

### Q6: 推理结果不准确

**原因**：模型输入尺寸不匹配（默认64x64，YOLOv8模型为640x640）。
**解决**：确认模型类型与推理参数匹配。mini-XCEPTION模型输入为64x64灰度图，YOLOv8为640x640三通道图。
