  - `wheelEvent`: 滚轮缩放
  - `paintEvent`: 自定义绘制标注框

### 3.3 模型量化模块

#### 3.3.1 量化原理
**动态量化 (Dynamic Quantization)**:
- 将模型权重从 FP32 (32位浮点) 转换为 INT8 (8位整数)
- 运行时动态计算激活值的量化参数
- **压缩比**: 11.7MB → 3.2MB (约73%压缩)

**量化公式**:
```
q = round(r / scale) + zero_point
r = (q - zero_point) * scale
```
其中 `r` 为浮点值，`q` 为量化后的整数值。

#### 3.3.2 实现代码
```python
from onnxruntime.quantization import quantize_dynamic, QuantType

# 动态量化
quantize_dynamic(
    model_input="best.onnx",      # 输入FP32模型
    model_output="best_quantized.onnx",  # 输出INT8模型
    weight_type=QuantType.QInt8   # 8位有符号整数
)
```

#### 3.3.3 量化引擎选择
| 引擎 | 支持情况 | 说明 |
|------|----------|------|
| ONNX Runtime | ✅ 支持 | 本项目使用的量化方式 |
| TensorRT | ❌ 未实现 | 需要NVIDIA GPU |
| PyTorch自带 | ❌ 未使用 | 本项目采用训练后量化 |

```
└─────────────────┴──────────────────────┘
```

#### 3.4.2 Python推理流程
```python
def inference_onnx(image_path, model_path):
    # 1. 加载模型
    session = onnxruntime.InferenceSession(model_path)
    
    # 2. 预处理 (letterbox缩放，归一化)
    img = cv2.imread(image_path)
    blob = preprocess(img)  # 640x640, /255.0
    
    # 3. 推理
    outputs = session.run(None, {"images": blob})
    
    # 4. 后处理 (NMS非极大值抑制)
    boxes, scores, classes = postprocess(outputs)
    
    return boxes, scores, classes
```

## 四、关键问题与解决方案

### 4.1 网络下载问题
**问题**: YOLOv8预训练权重 `yolov8n.pt` 需要从GitHub下载，国内网络不稳定。

**解决**:
1. 使用 HuggingFace 镜像 `hf-mirror.com` 手动下载
2. `train.py` 增加 `--weights` 参数支持本地权重文件
3. 检测权重文件存在性，不存在时给出清晰提示

### 4.2 量化模型兼容性
**问题**: OpenCV DNN 不支持量化后的 ONNX 模型 (`DynamicQuantizeLinear` 算子)。

**解决**:
1. UI增加检测：文件名含 `quantized` 时提示用户切换到 Python 模式
2. Python模式自动检测量化模型，回退到 `best.onnx`
3. 错误处理中加入友好提示信息

### 4.3 推理精度问题
**现象**: AN(angry)图片被识别为 disgust，置信度较低(45.6%)。

**原因分析**:
1. 数据集规模小 (JAFFE仅213张)
2. 训练轮数少 (30 epochs，可提升至100)
3. YOLOv8n 为轻量模型，精度有妥协

**改进方向**:
- 增加数据增强 (旋转、翻转、亮度调整)
- 使用更大的模型 (YOLOv8s/m)
- 增加训练轮数

### 6.2 可能被问到的问题

**Q1: 为什么选择YOLOv8而不是其他模型？**
- YOLOv8是one-stage检测器，速度快
- 有预训练权重，小数据集也能快速收敛
- Ultralytics框架封装完善，API简洁

**Q2: C++如何调用Python？为什么选择这种方式？**
- 使用 `QProcess` 启动Python子进程
- 优点：隔离性好，Python崩溃不影响主程序
- 缺点：通信开销稍大，但训练场景下可接受

**Q3: 量化后为什么不能用C++推理？**
- 量化模型使用 `ConvInteger` 等INT8算子
- OpenCV DNN 只支持标准FP32算子
- ONNX Runtime CPU provider 也未实现 `ConvInteger`

**Q4: 标注格式为什么选择YOLO格式？**
- YOLO是目标检测领域主流格式
- 与 Ultralytics 框架无缝兼容
- 归一化坐标，不受图像尺寸影响

**Q5: 如何提升模型准确率？**
- 增加训练数据 (数据增强、迁移学习)
- 调整超参数 (学习率、批次大小)
- 使用更大的模型 (YOLOv8s/m/l)
- 增加训练轮数
